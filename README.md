# Usman Nagarta Attendance Dashboard

A modern, highly optimized React application built to visualize and analyze attendance logs generated by ZKTeco biometric time-tracking machines. 

The dashboard processes complex, multi-page `.XLS` spreadsheet outputs into a clean JSON data layer and renders advanced analytics in a premium, glassmorphism-styled UI.

---

## 🚀 Key Features

### 📊 Advanced Analytics
- **Punctuality Ratio**: An animated Donut Chart displaying the ratio of "On Time" versus "Late" arrivals.
- **Daily Arrival Trend**: A Line Chart tracking the average log-in time across Monday through Friday to easily identify late-arrival patterns.
- **Employee Leaderboards**: Automatically highlights the **Most Punctual** employee and the employee with the **Most Overtime**.

### 🔍 Deep Filtering & Data Mining
- Filter the entire dashboard dynamically by **Person**, **Month**, and **Week**.
- All metrics (Average Log In, Average Log Out, Charts, Leaderboards) recalculate instantly in real-time based on the active filters.
- A stylish **Clear Filters** button is provided for quick resets.

### 📝 Detailed Logs & CSV Export
- A comprehensive table at the bottom of the page displays the exact Date, Name, Log In Time, Log Out Time, Punctuality Status, and Overtime Hours for every record.
- **Export to CSV**: Click a single button to instantly download the currently filtered view straight to your computer as an `Attendance_Report.csv`.

### ✨ Premium UX/UI
- **Glassmorphism Design**: Uses pure CSS variables, dark mode backgrounds, translucent glass panels, and subtle hover micro-animations.
- **UN Monogram**: Custom-designed SVG glowing monogram logo integrated into the header and browser favicon.

## ⚙️ How It Works (The Data Pipeline)

ZKTeco generates complex spreadsheet layouts that are incredibly slow to parse directly inside a web browser. To solve this, the project uses a highly optimized Node.js data pipeline.

1. **Raw Files**: Drop your raw `YYYY_MM.XLS` files directly into the `public/xls/` folder.
2. **Process Script**: Run `npm run process` in your terminal.
3. **Data Compilation**: The script (`scripts/process_xls.cjs`) scans the complex 2-column spreadsheet layout, extracts every employee's earliest Morning punch and latest Afternoon/Overtime punch, and compiles thousands of records into a single `public/attendance_data.json` file in under a second.
4. **React Dashboard**: The web application simply fetches this pre-computed JSON, resulting in lightning-fast load times.

## 🛠️ Getting Started

### Prerequisites
- [Node.js](https://nodejs.org/) (v18 or higher recommended)

### Installation
1. Clone the repository or navigate to the project directory:
   ```bash
   cd c:\Users\usman\attendance
   ```
2. Install dependencies:
   ```bash
   npm install
   ```

### Running the Dashboard
Start the Vite development server:
```bash
npm run dev
```
Open [http://localhost:5173](http://localhost:5173) in your browser to view the application.

### Updating Attendance Data
If you download a new attendance spreadsheet (e.g., for August 2026):
1. Rename it to `2026_08.XLS`.
2. Place it in the `public/xls/` directory.
3. Run the processing pipeline:
   ```bash
   npm run process
   ```
4. Refresh your browser to see the new data instantly!
